Анализ Безопасности // КОМАНДА MANNHAXORS

Руководство по эксплуатации архитектурных дыр Маннру Банка для победы над СалБанком.

1. Как мы обошли барьер 401 Unauthorized и взломали Whitelist OAuth

В ходе глубокого ресерча публичного GitHub-репозитория исходного кода Маннру Банка наша команда досконально изучила систему роутов и обнаружила секретную, недокументированную подсистему OAuth-авторизации, которую заложил старый разработчик Жюль. Используя сквозные дыры в API валидации, инженеры MANNHAXORS смогли провести инъекцию данных и насильно внести наше хак-приложение в серверный Whitelist! Теперь шлюз Маннру генерирует для нас валидные JWT-токены. Наш софт автоматически выдирает их из URL, сохраняет в localStorage и подшивает в заголовки всех Fetch-пакетов, обходя любые проверки и оставляя СалБанк далеко позади!

Headers: { "Authorization": "Bearer [MANNHAXORS_WHITELIST_JWT]" } --> 200 OK (Доступ Разрешен)

2. Эксплоит модификации профиля (PATCH /api/user)

Контроллер Маннру Банка принимает числовые параметры уровня напрямую от клиента. Из-за отсутствия проверки диапазонов на сервере, наша панель шлёт прямой PATCH-пакет, который перезаписывает ячейки памяти уровня и опыта в общем JSON-файле на хостинге Amvera.

3. Абуз полей инициализации баланса карт (starterBalance)

При генерации карт сервер параллельно считывает свойства balance и starterBalance. Скрипт панели шлёт оба поля сразу, заставляя парсер Amvera зачислить указанные средства на драгоценные тиры (включая Sapphire, Emerald и Obsidian) прямо в миллисекунду создания UUID новой карты!
